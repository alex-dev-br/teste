invalid-parameter

HTTP 400 (sugerido) Handler: ConstraintViolationException

Definição do tipo (estável)

Type (URI)
http://localhost:8081/erros/invalid-parameter/
Title (recomendado)
Invalid request parameter
Quando usar
Parâmetros de rota/consulta inválidos (@PathVariable/@RequestParam com @Validated).
Extensões
invalidParams[] ou errors[] com JSON Pointer

Exemplo de resposta (dinâmica)

{
        "type": "http://localhost:8081/erros/invalid-parameter",
        "title": "Invalid request parameter",
        "status": 400,
        "detail": "One or more parameters are invalid",
        "instance": "/api/v1/users?page=-1",
        "timestamp": "2025-10-27T12:34:56Z",
        "invalidParams": [{ "name": "page", "message": "must be ≥ 0" }],
        "errors": [{ "pointer": "/page", "detail": "must be ≥ 0" }]
        }

Contrato (RFC + extensões)

CampoTipoObrigatórioSemântica
type string (URI) Sim Identificador do tipo.
title string Sim Título canônico recomendado.
status number Sim HTTP status.
detail string Não Dinâmico.
instance string Não Dinâmico.
timestamp string Não Dinâmico.
invalidParams array Não Extensão.
errors array Não Alternativa.
